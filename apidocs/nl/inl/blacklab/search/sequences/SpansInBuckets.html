nl.inl.blacklab.search.sequences

Interface SpansInBuckets

  • All Known Implementing Classes:
    SpansInBucketsPerDocument, SpansInBucketsPerDocumentSorted

    public interface SpansInBuckets
    Interface to retrieve whole sequences of certain matches (in "buckets") instead of individual matches like with Spans. This is useful for efficiently processing sequences of related matches (i.e. fetch some content for all matches in one document). N.B. Note that in these classes, we avoid the term 'group' and 'grouping' because we already use these terms for the generic way of grouping spans (nl.inl.blacklab.search.grouping), while this is more focused on speed and efficiency of certain specific operations. Specifically, SpansInBuckets is designed to have random access to the contents of a bucket, but for efficiency's sake, only has sequential access to the buckets themselves. Also, SpansInBuckets uses subclassing instead of GroupIdentity objects to determine what goes in a bucket. This makes it easier to optimize. Note that SpansInBuckets assumes all hits in a bucket are from a single document.
